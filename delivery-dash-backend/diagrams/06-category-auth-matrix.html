Key rules

  • Vendor owner is inferred from the JWT (VendorId on the user). Request bodies never carry a VendorId for writes — it is set server-side.
  • VendorStaff can read vendor's categories (needed to pick category when adding products) but cannot create/update/delete.
  • Admin read on a vendor's product categories is for the vendor-detail page (audit / support). Admin never edits vendor menus.
  • Delete VendorCategory blocks if any vendor references it (HTTP 409). Admin must reassign first or set IsActive = false.
  • Delete Category succeeds even with products — affected products have their CategoryId nulled (uncategorized, not deleted).
  • Per-vendor cap: 50 categories max; 51st create returns 409.
