Category System — New Model (Class Diagram)

Two taxonomies: VendorCategory is global (admin-owned); Category is scoped per-vendor (vendor-owned). VendorType enum is removed.
classDiagram
    direction LR

    class VendorCategory {
        +int Id
        +string Name (unique)
        +string? Description
        +bool IsActive
        +DateTime CreatedAt
        +DateTime UpdatedAt
        --
        Admin-managed global taxonomy
        Seeded from VendorType enum values
    }

    class Vendor {
        +int Id
        +Guid UserId
        +string Name
        +string Description
        +string? ProfileImageUrl
        +TimeSpan OpeningTime
        +TimeSpan CloseTime
        +int VendorCategoryId  NEW
        --
        Type : VendorType  ← REMOVED
        Exactly one store category
    }

    class Category {
        +int Id
        +int VendorId  NEW
        +string Name (unique per vendor)
        +string? Description
        +int? SortOrder
        --
        ParentCategoryId ← REMOVED (flat)
        SubCategories[]  ← REMOVED
        Vendor-owned, flat
    }

    class Product {
        +int Id
        +int VendorId
        +int? CategoryId
        +string Name
        +decimal Price
        --
        Validator: Category.VendorId
        must equal Product.VendorId
    }

    class User {
        +Guid Id
        +string Email
        +Role Role
    }

    class Role {
        <<enumeration>>
        SuperAdmin = 0
        Admin = 1
        Vendor = 2
        Customer = 3
        Driver = 4
        VendorStaff = 5
    }

    VendorCategory "1" --> "0..*" Vendor : classifies
    User "1" --> "0..1" Vendor : owns (role=Vendor)
    Vendor "1" --> "0..*" Category : owns menu sections
    Vendor "1" --> "0..*" Product : sells
    Category "1" --> "0..*" Product : groups (nullable)
    User ..> Role : has role
Key changes:
  • VendorType enum → VendorCategory entity + FK on Vendor.
  • Catagory (typo) → Category; adds required VendorId, drops ParentCategoryId (flat).
  • Product.CategoryId stays nullable; service-layer validator ensures the referenced category belongs to the same vendor.
